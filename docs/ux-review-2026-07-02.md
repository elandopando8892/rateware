# Rateware UX Review — 2026-07-02

Full-app study performed on production (rateware.vercel.app) with live data
(55,766 approved rates, 645 vendors), signed in, desktop + mobile-width checks.
Every module was reviewed populated except Review Queue / Import (empty at review time).

Scale for each item: **Impact** (how much it improves daily use) / **Effort** (S/M/L).

---

## P0 — Trust & correctness

### 1. Evidence drawer headline shows the wrong lane
Selecting row `aeo.mx | Laredo, TX -> Mexico, DF | 1,968 USD | 2026-05-26` opens an
Evidence drawer titled **"Amarillo -> Augusta"**. Every other field matches the row,
so the headline lane label is derived incorrectly (likely a ZIP-prefix -> city lookup
misresolving, e.g. `739` and the MX destination). A wrong lane name at the top of the
"source of truth" drawer is a trust-killer for an audit tool.
**Where:** lane title composition in `src/rateware.js` (drawer render) and the
ZIP->market mapping used for display.
**Impact: very high / Effort: S-M** (find + fix the lookup; add fallback to raw
origin/destination strings when mapping confidence is low).

### 2. Semantic red is used for "informational" numbers
Two full-column red walls today: vendor completeness **51%** (red) on every pipeline
card, and **0/49 onboarding** (red) on every spreadsheet row. Red should mean
"action required now", otherwise the UI trains users to ignore red — which then
mutes real alerts (Failed uploads, Critical cells).
**Proposal:** 3-band thresholds (e.g. <35% red, 35-70% amber, >70% green) and neutral
styling for "not started" (0/49 in muted grey, red only when blocking an action).
**Where:** score/badge render helpers in `src/vendors.js`.
**Impact: high / Effort: S**

### 3. Governance timeline shows raw Kinde IDs
Audit entries read `kp_a636ecabd92d4fb3805bfd79...` — meaningless to a reviewer.
Show the user's name/email (claims are already in the JWT for the current user;
for others, store `actor_email` alongside `actor_id` at write time).
**Impact: medium-high / Effort: M** (touches write path + display).

## P1 — Information architecture

### 4. Bid Room renders the same 6-step workflow three times
On one screen: (a) "Bid Room Control" step cards Event->Lanes->Carriers->Preview->
Launch->Live bids, (b) a tab strip Launch/Dashboard/Event/Book/Invites/Live Bids,
(c) "Guided Bid Room Launch" with the SAME six steps again — plus **three**
"Create bid event" CTAs and three near-identical metric rows (Events/Lanes/
Invitations/Bids appears 3x in different forms). This is the most confusing screen
in the app: three competing mental models for one linear flow.
**Proposal:** ONE stepper owns the flow. Keep the tab strip as the only navigation;
delete the duplicate step rails; one primary CTA that changes with state
("Create bid event" -> "Add lanes" -> "Shortlist carriers" -> ...). One metric row.
**Where:** `rfx-events.html` + `src/rfx-events.js`.
**Impact: very high / Effort: M-L** (mostly deletion + moving state into the tabs).

### 5. Command Center is intake-centric; with a mature book it reads all-zero
With 55k approved rates, the dashboard shows: Pending 0, Failed 0, Open bids 0,
responses 0... The only life is "Approved rates 55,766". Daily users get no
intelligence from the home page.
**Proposal:** add a "Rate book health" band using data already in Supabase:
- **Quote freshness:** % of book older than 30/60/90 days (quote_date exists);
  55k rates quoted in May are aging right now.
- **Coverage:** lanes with 1 vendor vs 2+ (negotiation leverage), top 5 markets
  by rate count.
- **Recent activity:** last 7 days approvals/edits from the governance log.
Keep intake counters, but as a secondary strip.
**Where:** `src/dashboard.js` + 2-3 aggregate queries (mind statement timeouts —
see #9).
**Impact: very high / Effort: M-L**

### 6. Cross-module context strip is inconsistent and low-value
The 4-cell strip (Source files / Review queue / Rateware / Carrier CRM) repeats on
every page; on some pages it shows counts, on others literal dashes ("-"), and it
duplicates what the sidebar + page header already say. It costs ~48px on every
screen including dense workbenches.
**Proposal:** either make it a real global status bar (consistent counts + click-through
+ alert states) or remove it and reclaim the row. Recommend: remove on workbench pages,
keep on Command Center only.
**Impact: medium / Effort: S**

## P2 — Consistency & flow polish

### 7. Bilingual UI (EN chrome, ES content)
AI Copilot use-case cards and placeholder are Spanish ("Concentracion por carrier...",
"Dime los 30 carriers crossborder...") inside an English shell. Pick the product
language (or add a locale toggle); today it reads unfinished.
**Where:** strings in `src/business-intelligence.js`.
**Impact: medium / Effort: S** (decide + translate ~20 strings).

### 8. Hero-sized utility buttons
Full-width primary **Refresh** bars (Source Files, Vendors spreadsheet) and stacked
full-width "Download gaps" / "Import gap fixes" give maximum visual weight to the
least interesting actions, pushing data below the fold.
**Proposal:** auto-load data (it already does), demote Refresh to a small icon-button
in the toolbar row, group gap actions into a small split-button.
**Where:** `upload-history.html`, `vendors.html` + small CSS.
**Impact: medium / Effort: S**

### 9. AI Analyst geo map dies on the full dataset
"canceling statement due to statement timeout" — the density query over 55k rows
exceeds Supabase's statement timeout, so the flagship analytics view renders empty
with a tiny red error line.
**Proposal:** (a) server-side: pre-aggregated view or RPC with LIMIT/materialized
rollup; (b) UX: when it fails, show a real error state with a "Retry with smaller
scope" action instead of a silent empty map.
**Impact: high / Effort: M (UX part S)**

### 10. BI view switches freeze the tab
Switching Geo/Pivot/Copilot/Ranking repeatedly froze the renderer for 10-30s
(screenshot timeouts during review). Likely synchronous heavy DOM builds/queries on
tab switch. Defer data work until visible + show skeletons (Review Queue already has
a good skeleton pattern to copy).
**Where:** `src/business-intelligence.js` view-switch handlers.
**Impact: medium-high / Effort: M**

### 11. Vendor grid: six permanently-empty columns
Contact, Country, Services, Regions, Equipment, Certs are empty for all 629 vendors,
adding horizontal noise to every row. A Columns menu exists but defaults show all.
**Proposal:** default-hide columns that are >95% empty (with a "N hidden columns"
chip to restore), or promote the Columns menu state per saved view.
**Where:** `src/vendors.js` column defaults.
**Impact: medium / Effort: S-M**

### 12. Keyboard-hint strip is a wall of text
The staging/rateware helper strip lists ~14 shortcuts inline on every visit.
Collapse to a single "⌨ Shortcuts" chip that expands (localStorage-remembered).
**Impact: low-medium / Effort: S**

## Noted, not proposed (working as intended)
- Spreadsheet grids intentionally have no zebra striping (real cell-selection
  spreadsheet paradigm) — correct as is.
- The yellow floating button bottom-right is the **Vercel toolbar**, only visible to
  project members; end users don't see it. (Can be disabled in Vercel project
  settings if it annoys.)
- Review Queue skeleton loading state is the best loading pattern in the app —
  reuse it elsewhere.

## Suggested order
1. **#1 lane-title bug** (trust, small)
2. **#2 red thresholds** (trust, small)
3. **#8 + #6 + #12** quick sweep (visual calm, all S)
4. **#4 Bid Room de-duplication** (biggest structural win)
5. **#5 Command Center intelligence** (biggest daily-value win)
6. **#7, #9-UX, #11** consistency batch
7. **#3, #9-server, #10** need backend/infra touches
